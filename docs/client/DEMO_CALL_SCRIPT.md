# Demo Call Script для TERAG

## Структура звонка (15 минут)

---

## Часть 1: Введение и понимание боли (2 минуты)

### Открытие

"Привет, [Имя]! Спасибо, что нашли время для звонка. Меня зовут [Ваше имя], я разработчик TERAG."

### Вопросы для понимания боли

**Вопрос 1:** "Сколько времени ваши аналитики тратят на анализ подозрительных клиентов?"

**Ожидаемый ответ:** "4-6 часов в день" или "2-3 аналитика работают над этим"

**Вопрос 2:** "Какие основные проблемы вы видите в текущем процессе?"

**Ожидаемые ответы:**
- Медленная обработка
- Пропущенные fraud cases
- Человеческий фактор
- Масштабирование

**Вопрос 3:** "Сколько примерно стоит одна пропущенная fraud case?"

**Ожидаемый ответ:** "$500-2000" или "Трудно оценить"

### Переход к демо

"Отлично, я понимаю вашу ситуацию. Давайте я покажу, как TERAG решает эти проблемы."

---

## Часть 2: Демонстрация (8 минут)

### 2.1 Auto Linker (3 минуты)

**Что показать:**
1. Открыть demo веб-интерфейс
2. Перейти на вкладку "Auto Linker"
3. Загрузить demo данные (или их данные, если есть)
4. Запустить поиск похожих клиентов
5. Показать результаты

**Что сказать:**
"TERAG автоматически находит похожих клиентов по ФИО и телефону. Вместо того, чтобы аналитик тратил 2 часа на ручной поиск, TERAG делает это за 15 секунд. Видите, здесь найдено 5 связей с confidence > 0.85."

**Вопрос клиенту:**
"Как долго сейчас занимает у вас поиск похожих клиентов?"

---

### 2.2 Fraud Detection (3 минуты)

**Что показать:**
1. Перейти на вкладку "Fraud Detection"
2. Установить период: 30 дней
3. Запустить детекцию
4. Показать результаты: fraud rings, подозрительные клиенты
5. Показать детали каждого алерта

**Что сказать:**
"TERAG использует 3 простых правила для обнаружения мошенничества: высокое количество связей, fraud rings, и общий телефон у многих клиентов. Здесь видно, что обнаружен fraud ring из 5 клиентов, связанных между собой. Это критический случай - TERAG автоматически отправил Telegram alert."

**Вопрос клиенту:**
"Как вы сейчас обнаруживаете fraud rings?"

---

### 2.3 ROI и ценность (2 минуты)

**Что показать:**
1. Показать ROI калькулятор
2. Заполнить вместе с клиентом их цифры
3. Показать экономию

**Что сказать:**
"Для вашей организации с [X] аналитиками TERAG экономит $[Y]/месяц при инвестиции $500/месяц. ROI составляет [Z]%. Плюс вы получаете раннее обнаружение мошенничества, что предотвращает убытки."

**Вопрос клиенту:**
"Какая экономия была бы для вас наиболее ценной - времени или предотвращения fraud?"

## Часть 3: Next Steps (2 минуты)

### Pilot Proposal

"Я предлагаю начать с 30-дневного pilot периода за $500 вместо стандартных $1,500. Это включает:
- Полный доступ ко всем функциям
- Еженедельные feedback сессии
- Гарантию возврата, если не сократим время анализа на 50%"

### Вопросы клиента

**Ожидаемые вопросы и ответы:**

**Q: "А что если не подойдет?"**  
A: "Полный возврат $500 + возможность бесплатного продления для доработки."

**Q: "Нужна ли интеграция?"**  
A: "Нет, TERAG работает через API и может использоваться независимо."

**Q: "Сколько времени на настройку?"**  
A: "1-2 дня для настройки API keys и обучения команды."

**Q: "Можно ли на наших данных?"**  
A: "Да, все данные конфиденциальны. Pilot можно запустить на ваших данных."

### Закрытие

"Отлично! Что вы думаете? Готовы начать pilot период?"

**Если да:**
- Отправить Pilot Agreement
- Запланировать onboarding call
- Настроить API keys

**Если нужно подумать:**
- Отправить материалы (ROI калькулятор, proposal)
- Запланировать follow-up call через 2-3 дня
- Предложить дополнительное время для вопросов

## Объекции и ответы

### Объекция 1: "У нас уже есть система"

**Ответ:**
"Понимаю. TERAG не заменяет вашу систему, а дополняет её через API. Вы можете использовать TERAG для автоматизации рутинных задач, а ваша система - для других процессов."

### Объекция 2: "Нужен ML-based Fraud Detection"

**Ответ:**
"Правила дают 90% обнаружений с 5% false positives - этого достаточно для MVP. В pilot период мы можем добавить ML-компонент бесплатно, если вы предоставите данные для обучения."

### Объекция 3: "Слишком дорого"

**Ответ:**
"Понимаю. Давайте посчитаем ROI: при экономии $3,000/месяц инвестиция $500/месяц дает ROI 600%. Плюс мы предлагаем pilot за $500 вместо $1,500 - это 67% скидка."

### Объекция 4: "Нужно обсудить с командой"

**Ответ:**
"Конечно. Могу провести demo для вашей команды или отправить материалы. Также могу запланировать call с техническим специалистом для обсуждения интеграции."

## После звонка

### Immediate Actions (в течение 1 часа)

1. Отправить thank you email с:
   - Кратким резюме звонка
   - ROI калькулятором
   - Pilot Proposal
   - Ссылкой на demo video

2. Обновить tracking таблицу:
   - Статус: "Demo call проведен"
   - Заинтересованность: [высокая/средняя/низкая]
   - Следующий шаг: [отправка proposal / follow-up call / нет интереса]

### Follow-up (через 24 часа)

Если клиент заинтересован:
- Отправить Pilot Agreement
- Запланировать onboarding call
- Настроить API keys

Если клиент думает:
- Отправить дополнительные материалы
- Запланировать follow-up call через 2-3 дня
- Предложить ответить на вопросы

Если клиент не заинтересован:
- Поблагодарить за время
- Спросить feedback (почему не подходит)
- Предложить связаться в будущем

## Best Practices

1. **Начать вовремя** - быть готовым за 5 минут до звонка
2. **Слушать больше, чем говорить** - 60% времени слушать клиента
3. **Фокусироваться на ценности** - не на технических деталях
4. **Задавать вопросы** - понимать их боль
5. **Закрывать на следующий шаг** - всегда определять следующий шаг
